## 12.1 从单次任务到批量处理

学会将单次任务转化为批量处理，可以大大提高工作效率，让 Claude Code 发挥更大的价值。

## 记录成功的单次任务

### 记录的重要性

1. 经验积累：记录成功的经验，方便后续复用
2. 知识沉淀：将经验转化为知识，形成知识库
3. 效率提升：避免重复思考，直接使用经验
4. 质量保证：使用验证过的方法，保证质量

### 记录的内容

1. 任务描述：清晰描述任务的内容
2. 任务目标：明确任务的目标
3. 任务步骤：详细记录任务的步骤
4. 任务结果：记录任务的结果
5. 经验教训：总结经验教训

任务：生成产品文案

记录：

- 任务描述：为智能手表生成产品文案
- 任务目标：生成 200 字左右的产品描述文案
- 任务步骤：

  1. 提供产品信息
  2. 使用提示词模板生成文案
  3. 人工审核和调整
  4. 保存最终文案
- 任务结果：成功生成符合要求的产品文案
- 经验教训：提示词模板很有效，可以复用

### 重复性分析的方法 #

2. 模式识别：识别任务的共同模式
3. 需求分析：分析任务的需求变化
4. 价值评估：评估任务的价值

## 12.2 模板化与标准化

模板化和标准化是提高工作效率的重要手段。通过创建模板和建立标准，可以让任务更加规范，更加高效。

## 创建任务模板

### 模板的作用

1. 提高效率：避免重复思考，直接使用模板
2. 保证质量：使用验证过的模板，保证质量
3. 降低门槛：降低任务完成的门槛
4. 便于协作：统一的模板便于团队协作

### 模板的类型

1. 提示词模板：用于与 Claude Code 交互的模板
2. 文档模板：用于生成文档的模板
3. 流程模板：用于执行流程的模板
4. 数据模板：用于处理数据的模板

### 创建模板的步骤

步骤 1：分析任务

- 识别任务的共同特征
- 识别任务的可变部分
- 识别任务的固定部分

步骤 2：设计模板

- 设计模板的结构
- 设计模板的占位符
- 设计模板的说明

步骤 3：测试模板

- 使用模板完成任务
- 评估模板的效果
- 优化模板设计

步骤 4：完善模板

- 根据测试结果完善
- 添加使用说明
- 保存模板

### 模板创建示例 #

任务：生成产品文案

模板创建：

步骤 1：分析任务

- 共同特征：都是生成产品文案


## 开始接入 Crazyrouter

如果你准备把 Claude Code、国产模型或自己的应用统一接入 Crazyrouter，可以按这个顺序推进：

1. 到 [Crazyrouter 控制台](https://crazyrouter.com/console) 创建一个专用 API Token，并按项目或团队单独管理权限。
2. Claude Code 使用根域名：`https://cn.crazyrouter.com`；OpenAI 兼容 SDK 使用：`https://cn.crazyrouter.com/v1`。
3. 需要自动检查环境或快速写入配置时，使用 [Crazyrouter Claude Code 一键配置脚本](https://github.com/xujfcn/crazyrouter-claude-code)。
4. 调试失败时先看控制台日志，再核对 [API Endpoint 说明](https://docs.crazyrouter.com/api-endpoint)，重点检查 Base URL 是否多写了 `/v1`。

需要评估模型成本或选择不同模型时，可以先查看 [Crazyrouter 价格与模型页](https://crazyrouter.com/pricing)，再把常用模型加入 Token 白名单。
