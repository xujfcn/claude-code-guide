## 11.1 自定义工具的使用

Claude Code 支持使用自定义工具，可以扩展其功能，满足特定的需求。

## 什么是自定义工具

自定义工具是用户根据自己的需求创建的特定功能模块，可以扩展 Claude Code 的能力。

### 自定义工具的特点 #

1. 针对性：针对特定需求设计
2. 可复用：可以多次使用
3. 可配置：可以根据需要调整参数
4. 可集成：可以与其他工具集成

### 自定义工具的类型 #

1. 数据处理工具：处理特定格式的数据
2. 内容生成工具：生成特定类型的内容
3. 分析工具：进行特定的分析
4. 自动化工具：自动化特定流程

## 如何查找和使用现有工具

### 查找工具的方法 #

1. 官方文档：查看 Claude Code 的官方文档
2. 社区分享：在社区中查找用户分享的工具
3. 插件市场：在插件市场中查找工具
4. 搜索引擎：使用搜索引擎查找工具

### 使用工具的步骤 #

1. 安装工具：按照工具说明安装
2. 配置工具：根据需要配置参数
3. 使用工具：在 Claude Code 中调用工具
4. 评估效果：评估工具的效果
5. 优化调整：根据需要优化调整

## 简单工具的配置方法

### 配置示例：数据清洗工具

需求：需要一个工具来清洗 Excel 数据

配置步骤：

1. 定义功能

   - 删除重复行
   - 填充空白单元格
   - 统一日期格式
   - 统一数字格式
2. 创建提示词模板

```bash
请帮我清洗这个 Excel 数据：
- 删除重复行
- 填充空白单元格为"无"
- 统一日期格式为 YYYY-MM-DD
- 统一数字格式为保留两位小数
```

1. 保存为工具

   - 将提示词模板保存为工具
   - 命名为"数据清洗工具"
   - 设置快捷调用方式
2. 使用工具

   - 上传 Excel 文件
   - 调用"数据清洗工具"
   - 获得清洗后的数据

### 配置示例：文案生成工具 #

需求：需要一个工具来生成产品文案

配置步骤：

1. 定义功能

   - 生成产品描述
   - 生成广告文案
   - 生成社交媒体内容
   - 生成邮件内容
2. 创建提示词模板

```bash
请帮我生成[类型]文案：
- 产品名称：[产品名称]
- 主要特点：[特点1]、[特点2]、[特点3]
- 目标用户：[目标用户]
- 风格要求：[风格]
- 字数要求：[字数]
```

1. 保存为工具

   - 将提示词模板保存为工具
   - 命名为"文案生成工具"
   - 设置快捷调用方式
2. 使用工具

   - 提供产品信息
   - 调用"文案生成工具"
   - 获得生成的文案

### 智能助手的特点 #

## 11.3 自动化流程的设计

自动化流程是提高工作效率的重要手段。通过设计自动化流程，可以让 Claude Code 自动完成重复性任务，节省大量时间和精力。

## 识别可自动化的任务 #

### 自动化任务的特征 #

1. 重复性：需要重复执行的任务
2. 标准化：有明确标准和流程的任务
3. 规则明确：有明确规则的任务
4. 数据驱动：基于数据处理的任务

### 常见可自动化任务 #

1. 文档处理

   - 文档格式统一
   - 文档内容提取
   - 文档批量生成
2. 数据处理

   - 数据清洗
   - 数据分析
   - 报表生成
3. 内容创作

   - 文案生成
   - 报告撰写
   - 邮件回复
4. 流程执行

   - 会议纪要整理
   - 待办事项管理
   - 通知发送

## 设计自动化流程 #

### 流程设计原则 #

1. 简单性：流程应该简单易懂
2. 可靠性：流程应该稳定可靠
3. 可维护性：流程应该易于维护
4. 可扩展性：流程应该易于扩展

### 流程设计步骤 #

步骤 1：分析任务

- 识别任务的输入
- 识别任务的输出
- 识别任务的步骤

步骤 2：设计流程

- 设计流程的步骤
- 设计流程的顺序
- 设计流程的触发条件

步骤 3：实现流程

- 使用 Claude Code 实现流程

- 测试流程效果

步骤 4：优化流程

- 根据测试结果优化

- 提高流程效率

## 开始接入 Crazyrouter

如果你准备把 Claude Code、国产模型或自己的应用统一接入 Crazyrouter，可以按这个顺序推进：

1. 到 [Crazyrouter 控制台](https://crazyrouter.com/console) 创建一个专用 API Token，并按项目或团队单独管理权限。
2. Claude Code 使用根域名：`https://cn.crazyrouter.com`；OpenAI 兼容 SDK 使用：`https://cn.crazyrouter.com/v1`。
3. 需要自动检查环境或快速写入配置时，使用 [Crazyrouter Claude Code 一键配置脚本](https://github.com/xujfcn/crazyrouter-claude-code)。
4. 调试失败时先看控制台日志，再核对 [API Endpoint 说明](https://docs.crazyrouter.com/api-endpoint)，重点检查 Base URL 是否多写了 `/v1`。

需要评估模型成本或选择不同模型时，可以先查看 [Crazyrouter 价格与模型页](https://crazyrouter.com/pricing)，再把常用模型加入 Token 白名单。
