## 输入框使用

### 找到输入框

在 Claude Code 界面左侧的聊天窗口底部，你会看到一个输入框。这是你和 Claude Code 交流的主要区域。

### 输入你的问题

在输入框中输入你想问的问题或需求。例如：

- "你好，请介绍一下你自己"
- "帮我写一段产品介绍"
- "如何提高工作效率"

### 支持的操作

- 多行输入：按 Shift+Enter 可以换行，输入多行内容
- 粘贴文本：可以直接粘贴剪贴板中的文本
- 删除内容：使用 Backspace 或 Delete 键删除
- 全选：使用 Ctrl+A（Windows/Linux）或 Cmd+A（macOS）全选

## 发送消息

### 方法 1：按 Enter 键

输入完成后，直接按 Enter 键即可发送消息。

### 方法 2：点击发送按钮

在输入框右侧有一个发送按钮（通常是一个纸飞机图标），点击它也可以发送消息。

### 等待回复

发送消息后，Claude Code 会开始处理你的请求，通常几秒钟内就会给出回复。你会看到：

- 输入框上方出现"正在思考..."的提示
- 回复内容会逐字显示出来
- 回复完成后，提示消失

## 查看回复

### 回复内容

Claude Code 的回复会显示在对话历史区域，通常在左侧，带有 AI 图标。

### 回复格式

回复内容可能包含：

- 纯文本：普通的文字说明
- 列表：用项目符号或数字编号的列表
- 代码块：用特殊格式显示的代码（如果有）
- 表格：结构化的数据（如果有）

### 查看完整回复

如果回复内容较长：

- 可以向上滚动查看完整内容
- 或者在回复区域滚动查看

## 对话历史

### 查看历史对话

在输入框上方的区域，你可以看到你和 Claude Code 的所有对话历史：

- 最新的消息在最下方
- 可以向上滚动查看之前的对话
- 每条消息都有时间戳

### 继续对话

你可以随时在输入框中输入新的问题，继续对话：

- Claude Code 会记住之前的对话内容
- 可以引用之前的内容进行追问
- 可以转换话题开始新的讨论

### 清除对话

如果你想开始新的对话：

- 点击工具栏的"新建对话"按钮
- 或在设置中找到"清除历史"选项
- 这会清空当前对话历史，开始全新的对话

## 3.2 指定与操作文件

Claude Code 作为 VS Code 插件，可以直接访问和操作你工作区中的所有文件，无需手动上传。让我们学习如何高效地指定和操作文件。

## 理解工作区文件访问

Claude Code 直接集成在你的 VS Code 工作区中，这意味着：

- 无需上传：文件已经在你的工作区中，Claude Code 可以直接访问
- 实时同步：你对文件的任何修改都会立即反映在 Claude Code 的操作中
- 完整权限：Claude Code 可以读取、编辑、创建你工作区中的任何文件

## 指定文件的方式

### 方法 1：使用 @ 符号指定文件

这是最常用、最准确的方式。在聊天框中输入 `@` 后，会自动弹出文件列表供选择。

示例：

```bash
@main.js 请帮我优化这段代码的性能
```

使用技巧：

1. 输入 `@` 后，会显示当前工作区的所有文件
2. 可以输入文件名的一部分来过滤
3. 支持使用 Tab 键快速选择
4. 可以指定多个文件： `@file1.js @file2.js`

### 方法 2：直接使用文件路径

如果知道文件的完整路径，可以直接在对话中提及。

示例：

```bash
请帮我分析 src/components/Header.tsx 这个文件
```

### 方法 3：描述文件位置 #

通过描述文件的位置让 Claude Code 找到它。

示例：

```bash
请帮我修改 src 目录下的 utils.js 文件
```

## 文件操作类型 #

### 读取文件内容 #

让 Claude Code 读取并分析文件：

```bash
@config.json 请帮我读取并解释这个配置文件
```

### 编辑文件内容 #

修改文件中的特定内容：

```bash
@app.js 请将所有的 console.log 改为 logger.info
```

### 创建新文件 #

让 Claude Code 创建新文件：

```bash
请创建一个新的组件文件 src/components/Button.jsx，包含一个可点击的按钮
```

### 删除文件 #

删除不需要的文件：

```bash
请删除 test-old.js 这个文件
```

### 重命名文件 #

重命名文件：

```bash
请将 old-name.js 重命名为 new-name.js
```

## 批量文件操作 #

### 操作多个文件 #

可以同时指定多个文件进行操作：

```bash
@file1.js @file2.js @file3.js 请帮我在这些文件中添加错误处理
```

### 操作整个目录 #

对整个目录中的文件进行操作：

```bash
请帮我在 src/utils 目录下的所有文件中添加类型注解
```

## 文件搜索与定位 #

### 搜索文件内容 #

在文件中搜索特定内容：

```bash
请在项目中搜索所有包含 "TODO" 的代码行
```

### 查找特定函数 #

找到某个函数的定义：

```bash
请帮我在项目中找到 calculateTotal 函数的定义
```

### 查找引用 #

查找某个函数或变量的所有引用：

```bash
请帮我在项目中找到所有使用 useState 的地方
```

## 文件类型支持 #

Claude Code 主要处理代码文件和文本文件，对不同类型有不同的处理方式：

### 代码文件 #

- 支持格式：.js, .ts, .jsx, .tsx, .py, .java, .go, .rs, .cpp, .c, .php 等
- 处理方式：可以理解代码逻辑、进行重构、添加注释、修复 bug 等
- 示例：

```bash
@app.tsx 请帮我优化这个 React 组件的性能
```

### 配置文件 #

- 支持格式：.json, .yaml, .yml, .xml, .toml, .ini 等
- 处理方式：可以解析配置、修改设置、验证格式等
- 示例：

```bash
@package.json 请帮我添加一个新的依赖项
```

### 文本文件 #

- 支持格式：.txt, .md, .csv 等
- 处理方式：可以读取内容、进行文本分析、格式转换等
- 示例：

```bash
@README.md 请帮我更新文档中的安装说明
```

### 样式文件 #

- 支持格式：.css, .scss, .sass, .less 等
- 处理方式：可以优化样式、添加响应式设计等
- 示例：

```bash
@styles.css 请帮我将这个样式文件转换为响应式设计
```

## 高级技巧 #

### 使用相对路径 #

当文件较多时，使用相对路径更清晰：

```bash
请帮我修改 ./src/utils/helpers.js 中的 formatDate 函数
```

### 结合目录操作 #

先定位目录，再操作文件：

```bash
请帮我在 src/components 目录下创建一个新的 Card 组件
```

### 使用文件描述 #

当不确定文件名时，可以用描述：

```bash
请帮我找到并修改处理用户登录的那个文件
```

### 上下文感知 #

Claude Code 会根据你的操作历史自动理解上下文：

```bash
（之前操作了 UserService.js）
请帮我再添加一个用户注销的功能
```

### 代码片段引用 #

可以引用代码中的特定部分：

```bash
@App.tsx 请帮我优化第 45-60 行的渲染逻辑
```

## 注意事项 #

1. 文件路径：确保文件路径正确，使用 `/` 作为路径分隔符
2. 文件权限：确保 Claude Code 有权限访问和修改文件
3. 备份重要文件：在进行大规模修改前，建议先备份
4. 工作区范围：Claude Code 只能操作当前工作区内的文件
5. 文件大小：对于超大文件，建议分批处理

1. 提供更多细节：在请求生成文本时，提供更多细节可以获得更符合需求的结果

## 开始接入 Crazyrouter

如果你准备把 Claude Code、国产模型或自己的应用统一接入 Crazyrouter，可以按这个顺序推进：

1. 到 [Crazyrouter 控制台](https://crazyrouter.com/console) 创建一个专用 API Token，并按项目或团队单独管理权限。
2. Claude Code 使用根域名：`https://cn.crazyrouter.com`；OpenAI 兼容 SDK 使用：`https://cn.crazyrouter.com/v1`。
3. 需要自动检查环境或快速写入配置时，使用 [Crazyrouter Claude Code 一键配置脚本](https://github.com/xujfcn/crazyrouter-claude-code)。
4. 调试失败时先看控制台日志，再核对 [API Endpoint 说明](https://docs.crazyrouter.com/api-endpoint)，重点检查 Base URL 是否多写了 `/v1`。

需要评估模型成本或选择不同模型时，可以先查看 [Crazyrouter 价格与模型页](https://crazyrouter.com/pricing)，再把常用模型加入 Token 白名单。
