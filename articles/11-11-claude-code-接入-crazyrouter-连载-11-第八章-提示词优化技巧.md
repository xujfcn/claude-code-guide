## 第八章：提示词优化技巧

## 8.1 如何清晰表达需求

与 Claude Code 交流时，清晰表达需求是获得优质回复的关键。学会如何有效地提问，可以让 AI 更好地理解你的意图，给出更准确的答案。

## 具体明确的指令

### 避免模糊表达

使用具体、明确的指令：

模糊表达（不推荐）：

```bash
帮我写个文案
```

具体表达（推荐）：

```bash
请帮我写一段产品描述文案，产品是智能手表，主要特点包括健康监测、运动追踪、消息提醒，目标用户是年轻白领，字数在200字左右，风格要时尚现代。
```

### 提供完整信息 #

提供足够的信息，让 AI 理解你的需求：

信息不完整（不推荐）：

```bash
帮我分析这个数据
```

信息完整（推荐）：

```bash
请帮我分析这个销售数据，分析内容包括：
1. 总体统计（总销售额、平均销售额、最高/最低销售额）
2. 按产品分类统计
3. 按地区分类统计
4. 时间趋势分析
5. 发现数据洞察和改进建议
```

### 明确输出格式 #

明确告诉 AI 你想要的输出格式：

格式不明确（不推荐）：

```bash
帮我整理这些信息
```

格式明确（推荐）：

```bash
请帮我整理这些信息，输出格式为表格，包含以下列：姓名、部门、职位、入职日期。
```

## 提供必要的上下文 #

### 提供背景信息 #

提供相关的背景信息，帮助 AI 理解场景：

无背景信息（不推荐）：

```bash
帮我写个邮件
```

有背景信息（推荐）：

```bash
请帮我写一封邮件，背景是：客户对我们的产品提出了一个功能需求，我们需要回复客户，告知这个需求我们已经记录下来，会考虑在后续版本中实现，同时感谢客户的反馈。
```

### 说明使用场景 #

说明内容的使用场景，让 AI 调整表达方式：

无场景说明（不推荐）：

```bash
帮我写个介绍
```

有场景说明（推荐）：

```bash
请帮我写一个产品介绍，这个介绍将用于公司官网的产品页面，目标用户是潜在客户，需要突出产品的核心优势，语言要专业但不晦涩。
```

### 说明受众群体 #

说明内容的受众，让 AI 调整语气和风格：

无受众说明（不推荐）：

```bash
帮我写个说明
```

有受众说明（推荐）：

```bash
请帮我写一个操作说明，受众是没有任何技术背景的普通用户，语言要通俗易懂，步骤要详细，避免使用专业术语。
```

## 实用技巧 #

### 分步骤提问 #

对于复杂任务，分步骤提问：

一次性提问（不推荐）：

```bash
帮我分析这个数据，然后生成报告，再发送给相关人员
```

分步骤提问（推荐）：

```bash
步骤1：请帮我分析这个销售数据，生成统计结果。
步骤2：根据分析结果，帮我生成一份分析报告。
步骤3：帮我将报告转换为适合邮件发送的格式。
```

### 迭代优化 #

如果第一次结果不满意，可以迭代优化：

第一次提问：

```bash
请帮我写个产品文案
```

优化提问：

```bash
上次的文案有点太正式了，请帮我重新写一个，风格要轻松活泼一些，更贴近年轻人的表达方式。
```

### 多版本生成 #

要求生成多个版本，从中选择最好的：

单版本（不推荐）：

```bash
帮我写个文案
```

多版本（推荐）：

```bash
请帮我写3个不同风格的产品文案：
1. 专业风格
2. 幽默风格
3. 温馨风格
```

## 案例示例 #

### 案例：模糊 vs 清晰 #

对比模糊表达和清晰表达的效果：

场景：需要写一个会议通知

模糊表达：

```bash
帮我写个会议通知
```

AI 可能的回复：

```bash
好的，请问会议的主题、时间、地点、参会人员分别是什么？
```

清晰表达：

```bash
请帮我写一个会议通知，会议主题是"项目进度汇报"，时间是2024年1月20日14:00-16:00，地点是3楼会议室A，参会人员是项目组全体成员，会议议程包括：各模块进度汇报、问题讨论、下周计划。语气要正式、专业。
```

AI 的回复：

```bash
【会议通知】

各位同事：

兹定于2024年1月20日14:00-16:00在3楼会议室A召开项目进度汇报会。

会议议程：
1. 各模块进度汇报
2. 问题讨论
3. 下周计划

参会人员：项目组全体成员

请各位准时参加。

行政部
2024年1月18日
```

### 案例：无上下文 vs 有上下文 #

对比有无上下文的效果：

场景：需要回复客户邮件

无上下文：

```bash
帮我写个回复邮件
```

AI 可能的回复：

```bash
好的，请问需要回复什么内容？收件人是谁？
```

有上下文：

```bash
请帮我写一封回复邮件。背景：客户王先生昨天发邮件询问我们的产品是否支持批量导入功能。我们的产品目前不支持这个功能，但计划在下个版本中实现。请回复客户，告知这个情况，同时感谢他的反馈，并表示会通知他功能上线的时间。
```

AI 的回复：

```bash
尊敬的王先生：

您好！

感谢您对我们产品的关注和反馈。

关于您询问的批量导入功能，目前我们的产品暂不支持此功能。不过，我们非常重视您的需求，已将此功能列入开发计划，预计在下个版本中实现。

一旦该功能上线，我们会第一时间通知您。再次感谢您的宝贵反馈，这帮助我们不断改进产品。

如有其他问题，欢迎随时联系我们。

祝好！

[你的姓名]
[你的职位]
[公司名称]
```

## 小技巧 #

1. 使用结构化表达：用编号、列表等方式组织你的需求
2. 明确优先级：如果有多个要求，说明哪些是必须的，哪些是可选的
3. 提供约束条件：说明字数限制、格式要求等约束条件
4. 说明期望结果：描述你期望得到什么样的结果
5. 保持耐心：如果第一次结果不理想，可以调整需求，再次尝试
6. 记录好的提示词：将效果好的提示词记录下来，方便下次使用
